Release Notes

0.4.1

  • Updated installation documentation
  • Updated fixed version of SwiftProfobuf

0.4.0

Improvements

  • New API in the BroseConnectionManager canConnect() to check if the SDK has already initialized its Bluetooth stack
  • New API in the BroseConnectionManager resetConnectableBikes() to reset the cached connectable bikes

Changes

  • When a bike is not produced with the Brose Service Tool and has no valid bikeID, we are now publishing this bike as a BroseConnectableBike to be able to display the user a dedicated error that his bike is not correctly produced. The object has a new property isValid to be able to detect this case.

Bugfixes

  • Improved disconnect errors on connection handling
  • Fixed authentication problem with our backend
  • Fixed issue with duplicate data send to our backend
  • Fixed a potential crash in the Bluetooth communication

0.3.4

  • Fixed a potential crash with large data from the bike
  • Fixed data communication (receiving and sending data from our backend)

0.3.3

  • Chunking the telemetry data to prevent sending large data

0.3.2

  • If the connection process if failing, the error is now handled and published
  • The SDK is no more providing LiveData, if there are not drive unit data at all in it
  • When the BikeID in an advertisment is shorter than 9 characters, this advertisement will be not handled and the bike will not appear as a scanned bike
  • The driveUnitPower is now always positive (abs() is used)
  • More Logs were added on error cases

0.3.1

  • Fixed crash on disconnect of the bike
  • Improved disconnecting handling
  • Fixed crash on changing the thrustfactors
  • Exposing the drive unit power in Objective-C

0.3.0

Deployment using a XCFramework

The BroseSDK is delivered now as a XCFramwork instead of a manually builded fat framework. To use it with CocoaPods you need to have installed version 1.10.0 or newer.

In a XCFramwork it is currently not possible to have a class named equal to the module. Therefore we had to rename the class BroseSDK to BroseSDKManager.

Updated data model

The data model of the class BroseConnectedBike has changed to provide more information about the bike from the Service Tool.

Bugfixes

  • Fixed crash when Bluetooth were switched off during an active connection to a bike
  • Fixed Core Data crashes during an active connection

  • Smaller bugfixes
